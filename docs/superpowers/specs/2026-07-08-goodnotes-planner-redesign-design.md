# 굿노트 스타일 스터디 플래너 리디자인 설계

## 배경 및 목적

기존에 배포 중인 1페이지 인쇄용 스터디 플래너를, 아이패드 굿노트(GoodNotes)/노터빌리티에서 바로 쓸 수 있는 **하이퍼링크 내비게이션 다이어리**로 교체한다. 사용자가 수만휘/오르비 등 커뮤니티에 "직접 제작한 무료 배포 자료"로 자연스럽게 공유해 트래픽을 유입시키고, 애드센스 광고 수익을 늘리는 것이 목적이다.

기존 자료(네이버 블로그의 "365 플래너")를 참고하려던 시도는 저작권 문제가 있어 폐기하고, 사용자가 제공한 상세 프롬프트를 기반으로 완전히 새로운 구성을 제작한다.

## 범위

이번 구현은 사용자가 선택한 **핵심 구성(MVP)** 범위로 한정한다. 12개월 전체 반복 페이지, 성적 그래프, 오답노트, 단어장, 필기 노트 인덱스 등은 범위 밖이며, MVP 반응이 좋으면 후속 작업으로 추가한다.

### 포함 범위
- 표지, 나의 목표, 캘린더(12개월 미니 뷰), 시간표, 월간 달력(1개월 템플릿), 위클리 플랜(1주 템플릿), 데일리 로그(1일 템플릿), 습관 기록(1개월 트래커) — 총 8페이지
- 페이지 간 실제 클릭 가능한 PDF 내부 링크 (표지의 인덱스 탭, 캘린더의 각 월 → 월간 달력, 월간 달력 → 습관 기록)
- 가로형 4:3 비율(아이패드 굿노트 다이어리 비율)
- 기존 사이트(index.html, about.html)의 플래너 소개 문구를 "인쇄용"에서 "굿노트/디지털 다이어리용"으로 수정

### 범위 밖 (Out of scope)
- 12개월치 전체 반복 페이지 자동 생성
- 성적 추이 그래프, 오답노트, 단어장/암기 노트, 필기 노트 인덱스
- 표지 커버 여러 종류(심플/패브�릭 등) 옵션 제공 — 우선 1종만 제작
- 스티커팩(마스킹테이프, 하이라이트 아이콘) 데코 요소
- 손글씨 폰트 라이선스 확보 및 적용 (우선 시스템 고딕 폰트 사용)
- 판매용 제품화(스마트스토어, Etsy 등 마켓 대응, 목업 이미지 제작) — 이번 사이트는 무료 배포 전용

## 기술 접근

기존 프로젝트에서 이미 검증된 파이프라인을 그대로 재사용한다: HTML/CSS로 각 페이지를 작성하고, 헤드리스 Chrome(`--print-to-pdf`)으로 PDF를 생성한다. Chrome의 PDF 인쇄 기능은 문서 내 앵커 링크(`<a href="#섹션">`)를 실제로 클릭 가능한 PDF 내부 링크로 보존하므로, 별도의 PDF 라이브러리(reportlab 등) 없이 하이퍼링크 내비게이션을 구현할 수 있다.

### 페이지 규격
- `@page { size: 280mm 210mm; margin: 10mm; }` (4:3 비율에 가까운 가로형, A4 가로보다 살짝 넓게 조정)
- 각 섹션은 `<section id="...">` 로 감싸고, 인덱스 탭/링크는 해당 id로 이동하는 `<a href="#...">`로 구현

### 폰트/색상
- 폰트: 시스템 기본 고딕(`"Apple SD Gothic Neo", "Malgun Gothic"`) — 손글씨 폰트는 라이선스 확인 전까지 사용하지 않는다.
- 색상: 크림/버터 옐로우 배경 + 파스텔 브라운·그린 포인트 컬러. 기존 `style.css`의 색상 변수와는 별도로, 플래너 전용 팔레트를 `planner/planner.html` 내부 `<style>`에 정의한다 (기존 사이트 UI와 플래너 문서는 톤이 다르므로 분리).

## 파일 변경

- `planner/planner.html`: 기존 1페이지 인쇄용 마크업을 전면 교체 (8개 섹션)
- `downloads/study-planner.pdf`: 새 디자인으로 재생성
- `index.html`: 히어로 문구 및 특징 카드 3종 문구 수정 (굿노트 다이어리 반영)
- `about.html`: "인쇄해서 바로 쓰는" 관련 문구를 굿노트/디지털 다이어리 사용법으로 수정

## 테스트/검증 방법

- 로컬에서 `planner/planner.html`을 헤드리스 Chrome으로 PDF 변환 후 `pdftotext`로 텍스트 콘텐츠 확인
- 생성된 PDF를 브라우저 PDF 뷰어로 열어 표지의 인덱스 탭, 캘린더의 월별 링크를 클릭해 실제로 해당 섹션으로 이동하는지 확인
- 반응형/링크 확인이 끝나면 GitHub main에 push → Netlify 자동 재배포 → 배포된 사이트에서 다운로드 링크와 PDF 내용 재확인
